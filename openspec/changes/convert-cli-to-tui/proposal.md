# 将CLI界面转换为TUI界面

## 问题描述

当前应用中存在多个CLI（命令行）界面，这些界面在不同模式之间切换时会导致用户体验不一致，特别是在"任务列表->文件与操作"界面尝试打开文件时无反应，实际上是因为这些操作仍然调用CLI模式的外部程序。

## 当前CLI界面识别

1. **show_dashboard()** - 学习仪表板CLI显示
2. **show_achievements()** - 成就系统CLI显示
3. **open_in_vscode()** - 使用外部命令打开VSCode
4. **show_encouragement()** - 鼓励消息CLI显示

## 问题分析

1. **模式切换混乱**：TUI和CLI混合使用导致用户体验不一致
2. **终端状态冲突**：从TUI切换到CLI会导致终端状态异常
3. **外部依赖**：依赖VSCode等外部程序，降低可移植性
4. **输入阻塞**：CLI模式使用阻塞式输入，影响响应性

## 解决方案

### 1. 转换所有CLI界面为TUI
- 将`show_dashboard()`转换为TUI状态
- 将`show_achievements()`转换为TUI状态
- 实现文件内容查看器（替代VSCode打开）
- 将鼓励消息集成到TUI界面

### 2. 实现统一的文件查看器
- 在TUI内显示文件内容
- 支持常见文件类型（.md, .txt, .rs等）
- 提供基本的语法高亮（如果可行）
- 支持文件导航（返回上级目录）

### 3. 优化界面切换
- 确保所有操作都在TUI内完成
- 提供一致的导航体验
- 保持终端状态一致性

## 实施计划

1. **转换仪表板CLI** - 创建TUI版本的详细仪表板
2. **转换成就CLI** - 在TUI内显示成就信息
3. **实现文件查看器** - 替代VSCode打开功能
4. **集成鼓励消息** - 在适当位置显示鼓励信息
5. **测试验证** - 确保所有功能正常工作

## 预期效果

- 统一的用户体验，全部在TUI内完成
- 消除终端状态切换问题
- 提高应用的独立性和可移植性
- 更流畅的交互体验
