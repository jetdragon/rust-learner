# 学习指南

本指南将帮助你更好地使用这个 Rust 学习项目。

## 如何使用本项目

### 学习顺序

建议按照模块编号顺序学习，每个模块都建立在前一个模块的知识基础上：

```
module-01-basics → module-02-ownership → ... → module-10-project
```

### 学习流程

每个模块推荐的学习流程：

1. **阅读概念** → 阅读 `README.md` 了解核心概念
2. **运行示例** → 运行 `examples/` 中的代码，观察输出
3. **修改实验** → 修改示例代码，尝试不同的变化
4. **完成练习** → 做 `exercises.md` 中的练习题
5. **查看测试** → 运行 `cargo test` 验证答案
6. **综合项目** → 完成综合练习巩固知识
7. **自检评估** → 使用自检清单确认掌握程度

## 工作环境设置

### 安装 Rust

```bash
# Windows (下载并运行 rustup-init.exe)
# 或访问 https://rustup.rs/

# 验证安装
rustc --version
cargo --version
```

### 推荐工具

- **IDE**：VS Code + rust-analyzer 插件
- **终端**：Windows Terminal / PowerShell
- **编辑器配置**：启用格式化保存 (`rustfmt`)

## Cargo 常用命令

### 构建

```bash
# 构建所有模块
cargo build

# 构建特定模块
cargo build -p module-01-basics

# 发布构建（优化）
cargo build --release
```

### 运行

```bash
# 运行示例
cargo run -p module-01-basics --bin variables

# 运行并查看输出
cargo run -p module-01-basics --bin data_types
```

### 测试

```bash
# 运行所有测试
cargo test

# 运行特定模块的测试
cargo test -p module-01-basics

# 运行特定测试
cargo test test_temperature

# 显示测试输出
cargo test -- --nocapture
```

### 代码检查

```bash
# 格式化代码
cargo fmt

# 检查代码质量
cargo clippy

# 检查但不构建
cargo check
```

## 学习技巧

### 1. 动手实践

Rust 是一门实践性很强的语言，只看不练很难掌握：

```bash
# 每学一个概念就写代码验证
# 修改示例代码观察变化
# 尝试故意制造错误理解编译器提示
```

### 2. 理解错误信息

Rust 的编译器错误信息非常友好：

- **仔细阅读**：错误信息通常指出问题所在
- **建议修改**：编译器常给出修复建议
- **学习机会**：每个错误都是学习机会

### 3. 循序渐进

不要跳过基础：

- 所有权是 Rust 核心，必须理解
- 每个模块的自检清单要完成
- 练习题都要动手做

### 4. 记录笔记

在模块目录下创建自己的笔记文件：

```bash
# 在每个模块目录下
touch 我的笔记.md
```

### 5. 定期复习

使用 `进度.md` 追踪学习进度，定期复习之前的内容。

## 常见问题

### Q: 学习曲线很陡怎么办？

A:
- 不要着急，慢慢来
- 第一个模块最重要，打好基础
- 所有权的概念需要反复理解

### Q: 编译错误太多怎么办？

A:
- 逐个解决，不要被吓到
- 错误信息会告诉你问题
- 多使用 `cargo check` 快速检查

### Q: 记不住语法怎么办？

A:
- 正常现象，多写自然就记住了
- 可以准备一个速查表
- IDE 的代码提示很有帮助

### Q: 学完一个模块要多长时间？

A:
- 每人不同，大约 2-4 小时/模块
- 第一个模块可能会慢一些
- 质量比速度重要
