# 贡献指南

感谢你对本项目的关注！欢迎贡献新的学习内容、改进现有内容或修复问题。

## 如何贡献

### 1. 添加新的学习模块

如果想要添加新的学习主题：

1. **创建模块目录**：
   ```bash
   mkdir module-XX-topic
   cd module-XX-topic
   ```

2. **创建必要的文件**：
   - `Cargo.toml` - 包配置
   - `src/lib.rs` - 库入口文件
   - `README.md` - 概念讲解
   - `examples/*.rs` - 代码示例
   - `exercises.md` - 练习题
   - `tests/mod.rs` - 测试代码
   - `综合练习.md` - 综合项目
   - `自检清单.md` - 学习检查点

3. **更新工作空间**：在根 `Cargo.toml` 的 `members` 数组中添加新模块

4. **遵循模板**：参考 `模板/` 目录中的文档模板

### 2. 改进现有内容

- **修正错误**：发现内容错误时，直接修改并提交 PR
- **添加示例**：更多的示例代码有助于理解
- **优化解释**：让概念讲解更清晰易懂
- **增加练习**：更多练习题帮助巩固知识

### 3. 报告问题

如果在学习过程中遇到：

- 代码错误
- 内容不准确
- 解释不清楚
- 链接失效

请创建 Issue 描述问题。

## 内容规范

### 代码风格

- 遵循 Rust 标准风格：`cargo fmt`
- 通过 Clippy 检查：`cargo clippy`
- 添加清晰的中文注释
- 函数和变量命名使用英文

### 文档规范

- 使用简洁准确的中文
- 提供代码示例和运行结果
- 标注难度等级（简单/中等/困难）
- 包含常见问题和解答

### 练习题规范

每个练习应包含：
- 清晰的题目描述
- 具体要求列表
- 可选的提示（折叠显示）
- 验证命令

### 测试规范

- 为每个练习编写测试
- 测试应有清晰的断言
- 覆盖边界条件

## 提交流程

1. **Fork 项目**并创建分支
2. **进行修改**并确保所有测试通过
3. **提交更改**：使用清晰的提交信息
4. **推送分支**并发起 Pull Request

## 提交信息格式

```
<类型>: <简短描述>

<详细说明>

<关联 Issue>
```

类型示例：
- `feat`: 添加新功能
- `fix`: 修复问题
- `docs`: 文档更新
- `style`: 代码格式
- `test`: 添加测试
- `refactor`: 代码重构

## 代码审查

所有 PR 需要通过：

- [ ] 代码风格检查（`cargo fmt`）
- [ ] Clippy 检查（`cargo clippy`）
- [ ] 所有测试通过（`cargo test`）
- [ ] 示例可运行（`cargo run`）
- [ ] 文档完整准确

## 许可证

贡献的内容将使用与项目相同的 MIT 许可证。
